上下文工程：决定 Agent 上限的隐藏变量

很多人以为 Agent 不稳定是模型不行。但真正的瓶颈，往往是上下文质量。

同一个模型，给它精心组织的上下文，和塞给它一堆原始日志，效果天差地别。

什么是上下文工程

上下文工程，本质是一个信息选择问题：在有限的 token 预算内，决定放什么、不放什么。

上下文的质量，决定了 Agent 的能力上限。

三个核心动作

  • 压缩：把长对话压成关键事实
  • 检索：按需拉取相关知识，而非全量塞入
  • 排序：把最相关的放在注意力最集中的位置

关键参数示例：max_tokens=8000、top_k=5。

一个最小实现

def build_context(query, memory):
    related = retrieve(query, top_k=5)
    return compress(memory) + related
